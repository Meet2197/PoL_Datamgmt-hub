Gitlab TUD

Gitlab TUD is a powerful platform for code management and collaboration, providing a Git repository for version control, issue tracking, and continuous integration/continuous deployment (CI/CD) pipelines. It is the university's central service for managing research software and scripts.

  • Version Control: Track changes to your code over time, making it easy to revert to previous versions and collaborate without conflicts.
  • CI/CD: Automate the building, testing, and deployment of your code, ensuring consistent and reproducible results.
  • Collaboration: Work on projects with multiple team members, manage issues, and review code changes efficiently.

FAQs & Helpful Info for Researchers

What is Git, and why is it important for my research?

Git is a version control system that tracks changes to your files. For researchers, this means you can save different versions of your code, scripts, or thesis chapters, making it easy to see what changed and who made the change. It's crucial for reproducibility and collaborative work.

How does Gitlab TUD help with code reproducibility?

By hosting your code in a Git repository, you create a public or private record of every change. This means that anyone can access the exact version of the code used for a specific publication, ensuring that the computational aspects of your research are fully reproducible.

Can I use Gitlab TUD for data as well as code?

While Git is designed for code, Gitlab can handle smaller data files. For large datasets, it is better to use dedicated data repositories like OpARA TUD, Zenodo, or Figshare and link to them from your Gitlab project.

Is this service only for TU Dresden members?

Yes, Gitlab TUD is an internal service provided by the university's IT center. It is specifically for members of the TU Dresden community to manage their research projects and is integrated with university accounts.
